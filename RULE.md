# Agent 工作流程規範 (RULES.md)

作為本專案的 AI 開發助手，你在執行任何開發任務時，必須嚴格遵守以下 S.O.P (標準作業流程)：

## 1. 測試驅動與驗證 (Testing)
- 本專案使用 `pytest` 進行單元測試。
- 每當新增一個功能（例如：解析字串、產生 MOZE URL），你必須先在 `tests/` 目錄下寫好對應的測試案例。
- 寫完程式碼後，**你必須自行在終端機執行 `pytest`**。
- 如果測試失敗，你必須自行閱讀錯誤訊息並修正程式碼，直到測試完全通過為止，才能進行下一步。

## 2. 自動化版本控制 (Git)
- 每當一個功能模組開發完成，且 **測試通過後**，你必須自動執行 Git 操作。
- 執行 `git add .` (或指定檔案)。
- 使用 Conventional Commits 規範撰寫 Commit Message，例如：
  - `feat: 新增字串解析功能與 MOZE URL 產生器`
  - `test: 新增字串解析的 pytest 案例`
  - `fix: 修正 URL 中文字元未 urlencode 的問題`

## 3. 文件同步更新 (Documentation)
- 你必須保持 `README.md` 的即時性。每當引入了新的環境變數 (.env) 或新的套件，必須立刻更新 README 裡的安裝與設定說明。
- （可選）在專案根目錄維護一個 `CHANGELOG.md`，簡述你剛才完成了哪些功能。
